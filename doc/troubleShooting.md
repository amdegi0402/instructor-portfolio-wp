# トラブルシューティング記録：お問い合わせフォームのメール未達

## 事象
Contact Form 7でお問い合わせフォームを設置し、テスト送信を行ったが、管理者メールアドレスにメールが届かなかった。

## 原因の切り分け
1. フォーム自体の送信は正常に完了していることを確認（エラー表示なし）
2. 管理者メールアドレスの設定を確認 → 問題なし
3. 迷惑メール（スパム）フォルダを確認 → 該当メールなし
4. 上記から、フォーム側の問題ではなく、**サーバー側のメール送信機能（PHPのmail関数によるデフォルト送信）に起因する未達**と判断

## 対応方針
レンタルサーバーのデフォルトのメール送信方式は、迷惑メール判定を受けやすく到達率が低い傾向があるため、Gmail（Google Workspace）のSMTPサーバーを経由して送信する方式に変更することとした。

## 実施手順

1. WP Mail SMTPプラグインをインストール・有効化
2. Mailerとして「Google Workspace / Gmail」を選択
3. Google Cloud Consoleで新規プロジェクトを作成
4. Gmail APIを有効化
5. OAuth同意画面（Google Auth Platform）にてアプリ情報を設定
6. OAuthクライアントID（ウェブアプリケーション）を作成
   - WordPress側に表示されたリダイレクトURIを、Google Cloud Console側の該当欄に登録
7. 発行されたクライアントID・クライアントシークレットをWP Mail SMTPの設定画面に登録

## 発生した追加のエラーと対処

**エラー内容**：Gmailとの連携時に「アクセスをブロック：アプリがGoogleの審査プロセスを完了していません（エラー403: access_denied）」が発生

**原因**：作成したOAuthアプリが「テスト」ステータスのままで、未登録のGoogleアカウントからのアクセスが制限されていたため

**対処**：Google Cloud Consoleの「対象」設定にて、使用するGmailアドレスを「テストユーザー」として追加登録。再度連携を実行し、認証成功。

## 結果
WP Mail SMTPの「メールテスト」機能、および実際のお問い合わせフォームからのテスト送信で、管理者メールアドレスへの到達を確認。問題解決。

## 学んだこと
- メール送信の不具合は、フォーム側・サーバー側・送信経路（SMTP）のどこに原因があるか、切り分けて特定することが重要
- OAuth認証まわりのエラーは、アプリの公開ステータス（テスト／本番）に起因することが多く、エラーメッセージを正確に読み解くことで解決の糸口が見つかる
